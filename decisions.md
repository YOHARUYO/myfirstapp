> 마지막 업데이트: 2026-04-16

---

## 프로젝트 개요

**목적:** 대면 회의를 자동으로 기록·요약하고 Slack으로 전송하는 개인용 회의록 자동화 앱

**사용자:** 혼자 사용 (개인 로컬 환경)
**회의 형태:** 대면 회의

---

## 기술 스택

| 영역 | 결정 |
|------|------|
| 프론트엔드 | React 18 + TypeScript (Tiptap, Zustand, Tailwind CSS) |
| 백엔드 | Python + FastAPI |
| 실시간 STT | Web Speech API (브라우저 내장) |
| 정밀 STT | 로컬 Whisper (무료) → 필요 시 OpenAI Whisper API로 전환 |
| AI 요약 | Claude API |
| 히스토리 저장 | JSON 파일 (메타데이터 구조화 포함, 아래 참고) |
| Whisper 모델 | medium (로컬, M3 MacBook Pro 14" 기준 1시간 회의 ~5-8분 처리) |
| Slack 연동 | Slack Bot API (메시지 전송 + Canvas 작성) |
| 캘린더 연동 | Google Calendar API |

## 핵심 설계 결정 사항

### Whisper 재처리 + 사용자 수정 보존
- 실시간 녹음 시 Web Speech API로 즉시 텍스트 표시
- 녹음 완료 후 Whisper로 오디오 재처리
- 타임스탬프 기반으로 사용자가 수정한 구간은 잠금 처리
- 겹치는 구간이 있으면 보수적 규칙 적용 (잠금 우선)
- 미수정 구간만 Whisper 결과로 교체

### JSON 히스토리 저장 구조
- 회의 1건 = JSON 파일 1개
- 각 파일에 메타데이터 구조화 저장 포함:
  - 날짜, 제목, 참여자, 소요 시간
  - 키워드 목록 (트렌드 리포트용)
  - 전사 텍스트 원본
  - AI 요약 결과
  - 액션 아이템 목록
- SQLite 없이 JSON 파일 집계로 트렌드 리포트 생성
- 추후 데이터가 많아지거나 검색 성능 필요 시 SQLite 인덱스 추가 검토

## UI/화면 구성 결정 사항

- 화면 이동 방식: wizard 스타일 (단계별 순서 진행)

  - 돌발 회의: 제목만 입력 후 바로 녹음 시작, 나머지는 편집 화면에서 사후 입력
  - 참여자 주소록 및 자주 쓰는 장소 저장 기능 포함

## UI 설계 결정 사항

### 전체 화면 구조 (wizard 유지)

| 단계 | 화면 | 비고 |
|------|------|------|
| 1 | 홈 / 시작 | 신규 회의 시작, 히스토리 진입, 복구 배너 |
| 2 | 회의 정보 입력 | 템플릿/빠른 시작, 입력 소스 선택, 마이크 권한 요청 |
| 3 | 녹음 & 실시간 전사 (녹음 경로) / 업로드 (업로드 경로) | 모드별 UI 분기 |
| 4 | Whisper 후처리 | 두 경로 공통, 잠금 구간 병합 |
| 5 | 회의록 편집 | 중요도 태깅 본격 편집, AI 1차 태깅 |
| 6 | 요약 & 액션 아이템 | 중요도 기반 Claude 요약 |
| 7 | 전송 & 저장 | Slack 전송, 로컬 저장, JSON 히스토리 기록 |
| 별도 | 히스토리 화면 | 목록/검색/상세, 5·6단계 레이아웃 재사용 |
| 별도 | 설정 화면 | 템플릿, 주소록, API 키, Whisper 모델 |

### 경로 분기 원칙
- **2단계에서 모드 선택** → 3단계 UI 분기 → **4단계 이후 재수렴**
- 녹음 경로 3단계: 실시간 전사 (핵심 UX, 가장 복잡)
- 업로드 경로 3단계: 파일 드롭 + 진행률 (최소 UI, 패스스루)
- 4단계: 공통 Whisper 처리 (업로드 경로는 잠금 병합 로직 불필요)

### 3단계 (녹음 & 실시간 전사) 상세 설계

#### 레이아웃 (세로 스택)
```
[ [🏠] wizard stepper                        ]
[ 상태바 (sticky top)                         ]
[   dot / 상태 / 타이머 / 레벨 / 언어         ]
[ 회의 정보 요약 (편집 가능, 접기, 기본 접힘)  ]
[   마이크 민감도 슬라이더 (패널 안)           ]
[                                             ]
[ 실시간 전사 영역 (스크롤)                   ]
[                                             ]
[ ┌────────┐                                  ]
[ │● 시작  │  ← FAB (좌하단 고정)              ]
[ └────────┘                                  ]
[ [← 이전 단계]             [다음 단계 →]    ]
```

- **상태바 (sticky top)**: 녹음 dot(idle/pulse) + 상태 텍스트 + 타이머 + **최소형 마이크 레벨 미터** + 현재 언어. 스크롤해도 항상 상단에 고정 — 녹음 중인지 즉시 확인 가능
- **회의 정보 요약**: 2단계 입력값을 **편집 가능** 상태로 노출(회의 중 참여자/장소 변동 반영), 접기 가능, **기본 접힘** (녹음 시작 후 참조 빈도 낮음, 전사 영역 최대화)
  - 접힌 상태에서는 패널 헤더만 보임 → 전사 영역 최대화
  - **마이크 민감도 슬라이더**: 회의 정보 패널 안, 언어 설정 아래에 배치
    ```
    🎤 마이크 민감도  [━━━━━━━●━━━] 높음
    ```
    - 범위: 0.5x ~ 3.0x, 기본값: 설정 화면의 디폴트값 (초기 1.0x)
    - Web Audio `GainNode`로 오디오 스트림을 증폭한 뒤 MediaRecorder에 전달
    - 패널이 접힌 상태에서는 보이지 않아 녹음 화면 정돈 유지
    - 녹음 중에도 실시간 조절 가능 (GainNode.gain 값 즉시 반영)
- **전사 영역**: 가장 큰 영역 (스크롤 가능)
- **녹음 컨트롤 FAB (Floating Action Button)**: **모든 상태에서 상시 표시**, 콘텐츠 영역 **좌하단**에 고정
  - **위치**: 하단 네비게이션 바([← 이전][다음 →]) 바로 위, **좌측** 배치
    - 하단 네비 바와 `mb-4`(16px) 간격
    - 콘텐츠 영역 좌측 시작점에 맞춤 (좌측 padding과 정렬)
  - **크기**: 56x56px 원형, `shadow-lg`
  - **상태별 외관**:
    | 상태 | 색상 | 아이콘 | 라벨 |
    |------|------|--------|------|
    | idle | `bg-success` (초록) | Mic | 시작 |
    | recording | `bg-recording` (빨강) | Square | 중지 |
    | post_recording | `bg-success` (초록) | Mic | 재개 |
  - 스크롤과 독립적 — 전사가 길어져도 항상 보임

#### 3단계 내부 상태
| 상태 | dot | 타이머 | 변형 슬롯 | 중지 | 다음 단계 |
|------|-----|--------|----------|------|----------|
| 1. idle (녹음 전) | idle | 00:00 | `● 녹음 시작` | — | 비활성 |
| 2. recording | pulse | 진행 | `■ 중지` | — | 비활성 |
| 3. post-recording (녹음 후 검토) | idle | 정지(최종) | `● 녹음 재개` | — | **활성** |

- 컨트롤 바는 **단일 변형 슬롯** (상태에 따라 아이콘·라벨·동작 변화), 일시정지 없음
- 다음 단계 버튼: 하단 우측 고정, 상태 3에서만 활성, 편집 세션 진행 중엔 자동 비활성 (옵션 α)

#### 실시간 전사 표시 (하이브리드)
- 평상시 연속 텍스트처럼 읽히되 내부는 **문장 블록 구조** 유지
- 블록 호버 시 타임스탬프·수정 버튼 노출
- **final / interim 구분**: interim은 마지막 final 옆이 아닌 **별도 회색 블록** 으로 표시
- **타임스탬프**: 녹음 중엔 **상대시각만** (예: `00:05:12`), 히스토리 화면에서 **절대/상대 토글** 제공

#### interim 블록 클릭 → 강제 확정
- 녹음 중(상태 2) 회색 interim 블록을 **싱글 클릭**하면 현재까지의 텍스트를 **즉시 final 블록으로 확정**하고, 다음 음성부터 새 블록으로 전사 시작
- 용도: Web Speech가 문장을 너무 길게 이어붙일 때 사용자가 원하는 시점에서 블록을 끊을 수 있음
- 동작 원리: `recognition.stop()` → 현재 interim이 final로 확정 → `onend` 핸들러가 자동 재시작
- **final 블록에는 적용 안 됨** — final 블록 싱글 클릭은 아무 동작 없음 (더블 클릭이 편집 진입)
- 녹음 중이 아닌 상태(idle, post-recording)에서는 interim 블록 자체가 없으므로 해당 없음

#### 블록 구조 (좌측 슬롯)
```
[화자 슬롯] [중요도 바] [00:00:14] 본문…
                        [✎]
```
- **화자 슬롯**: MVP에서는 빈 원 (화자 미지정), 5단계 편집 화면에서 수동 드롭다운 태깅, 3phase에 pyannote 자동 채움
- **중요도 바**: 색상 세로 바 (상=빨강, 중=노랑, 하=회색, 최하=투명, 미지정=없음)
- **수정 표시 (✎)**: **실제 텍스트가 변경된** 블록에만 표시. 편집 모드에 진입했더라도 내용을 바꾸지 않고 나오면 수정 표시 없음. Whisper 재처리 잠금 기준과 동일
  - **표시 방식 (A+C 복합)**: 타임스탬프 색상 변경 + 타임스탬프 바로 아래 연필 아이콘
    - 수정된 블록의 타임스탬프: `text-text-tertiary` → **`text-primary`** (파란색으로 변경)
    - 타임스탬프 바로 아래에 `✎` 아이콘(Pencil, 10px, `text-primary`) 표시
    - 타임스탬프와 아이콘은 **고정 폭 컨테이너**(w-12) 안에서 세로 스택 → 본문 텍스트 영역의 가로/세로 크기에 영향 없음
    - 미수정 블록: 타임스탬프만 표시 (회색), 아이콘 자리는 빈 공간 (높이 확보 안 함 → 레이아웃 shift 없음)
  ```
  수정됨:    [00:05:12]  본문 텍스트…
             [✎]
  미수정:    [00:05:12]  본문 텍스트…
  ```

#### 인라인 편집 UX
| 동작 | 트리거 |
|------|--------|
| 편집 진입 | 블록 본문 **더블클릭** (or 키보드 포커스 시 Enter) |
| 확정 | Enter / 블록 밖 클릭 |
| 취소 | Esc |
| 수정 중 녹음 | 해당 블록만 잠금, 녹음은 계속 진행, 편집 블록 위치로 스크롤 고정 |

- 중지 버튼은 녹음만 멈춤 → 3단계 이탈 아님 → 편집 세션 진행 중이어도 모달 불필요
- 단계 이탈 순간은 오직 [다음 단계 →] 클릭 시점

#### 중요도 태깅
- **4레벨**: 상/중/하/최하 + 미지정
- **단계별 허용**:
  - 녹음 중(상태 2): **숫자 단축키만** 지원 (녹음 몰입 유지), 집중 방해 최소화
  - post-recording(상태 3): 단축키 + UI 조작 모두 가능
  - 5단계: 본격 태깅 편집 (AI 1차 태깅 포함)
- **AI 1차 태깅**: 5단계 진입 시 실행, 미태깅 블록에만 적용. **사용자 수동 태그는 항상 우선**(AI가 덮어쓰지 않음)
- **조작 방법**:
  - 마우스: 중요도 바 클릭 → 가로 4색 팝오버 → 클릭 선택
  - 키보드: ↑/↓ 로 블록 포커스 이동 → **1/2/3/4 = 상/중/하/최하, 0 = 미지정**
  - 녹음 중에도 같은 1/2/3/4/0 키 체계 유지 (모든 단계 통일)
- **포커스 트리거**: 편집 진입(더블클릭)과 hit area 분리
  - 마우스: 중요도 바 클릭만이 진입점
  - 키보드: 방향키 네비게이션

#### 녹음 재개 / 빈 구간 처리
- 상태 3에서 [● 녹음 재개] → 상태 2 복귀, 새 블록이 기존 전사 뒤에 이어붙음
- **무제한 반복 허용** (실제 회의에서 자주 발생)
- **빈 구간 표시**: 타임스탬프는 실시간 누적(옵션 C) + `⏸ N분 경과` 구분선으로 시각 분리

#### 이탈 / 사고 대응
| 시나리오 | 대응 |
|---------|------|
| 마이크 권한 거부 | **2단계에서 미리 권한 요청** |

#### 오디오 보존

### 5단계 (편집 화면) 상세 설계

#### 5단계 ↔ 6단계 분리 확정
- 이유: **AI 요약은 중요도 태깅 결과를 입력으로 써야 하므로** 태깅 완결(5단계) → 요약 생성(6단계) 순차가 자연스러움
- wizard의 단계별 역할 분리 원칙과도 일치

#### 레이아웃 (단일 컬럼, 옵션 A)
```
[ [🏠] wizard stepper                                ]
[ 메타데이터 영역 (편집 가능, 접기, 기본 펼침)       ]
[ 미니멀 툴바 (sticky)                               ]
[                                                     ]
[ 전사 영역 (3단계 블록 구조 재사용)                 ]
[                                                     ]
[ [← 이전 단계]              [다음 단계 →]           ]
```

- 3단계와 레이아웃 연속성 유지 (블록 구조·색상 바·화자 슬롯 동일)
- 보조 정보(태그 범례·통계 등)는 상시 노출 안 함 → 필요 시 툴바 버튼으로 호출

#### 미니멀 툴바 구성
| 구분 | 기능 | 설명 |
|------|------|------|
| 코어 | 일괄 치환 | 상단 고정 검색/치환 바 호출 |
| 코어 | 핵심/전체 토글 | **상+중만** 표시 ↔ 전체 표시 |
| 코어 | Undo / Redo | 편집 이력 복구 |
| 확장 | 검색(찾기만) | 본문 내 탐색 |
| 확장 | 단축키 도움말 | 키보드 치트시트 |
| 확장 | AI 재태깅 | 미태깅 블록에만 재실행 |

#### 중요도 태깅 (5단계 본격)
- **AI 1차 태깅**: 4단계(Whisper 후처리) 후반에 자동 실행, 사용자 체감상 "4단계 안에서 처리되는 중"
  - 사용자 인지로는 4단계 로딩 안에 포함, 5단계 진입 시 이미 태깅된 상태
- **사용자 수정 태그는 항상 우선** — AI가 덮어쓰지 않음
- **재실행 허용 범위**: 미태깅 블록에만 재실행 (툴바의 "AI 재태깅" 버튼)
- **AI 재태깅 진행 표시**: 버튼 클릭 → 버튼이 **로딩 스피너 + "태깅 중…"** 텍스트로 전환 (버튼 비활성), 완료 시 원복 + 토스트 "N개 블록 태깅 완료"
- **"핵심만 보기" 정의**: 상 + 중 (6단계 요약 생성 입력 기준과 동일) → *"지금 보이는 것이 그대로 요약된다"* 멘탈 모델
- **다중 선택 태깅**: MVP 미지원
- **조작 방식**: 3단계와 동일 (마우스: 색상 바 클릭 → 가로 4색 팝오버 / 키보드: ↑↓ 포커스 → 1/2/3/4/0)

#### 전사 편집 방식
- **블록 구조 유지 + 병합·분할 허용** (옵션 2)
- **편집 진입**: 블록 본문 **더블클릭** (3단계와 동일) 또는 포커스 상태에서 **Enter**
- **편집 중 키 할당**
  | 키 | 동작 | 비고 |
  |---|------|------|
  | Enter | 편집 확정 | |
  | Esc | 편집 취소 | |
  | 블록 밖 클릭 | 자동 확정 | |
  | **Shift + Enter** | **블록 내 줄바꿈** (개행 삽입) | 일반 텍스트 에디터 관례와 동일 (Slack, Notion, Google Docs) |
  | **Ctrl + Enter** (Mac: Cmd+Enter) | **커서 위치에서 블록 분할** | 의도적 동작이므로 수정자 키 조합 |
  | Backspace (블록 시작) | 위 블록과 병합 | **첫 번째 블록에서는 무시** (병합 대상 없음) |
  | Delete (블록 끝) | 아래 블록과 병합 | |
- **분할 트리거**: Ctrl+Enter(Mac: Cmd+Enter)만. **호버 메뉴로 분할은 제공 안 함** (메뉴 클릭이 커서 위치를 전달하지 못하는 근본적 UX 문제)
- **병합 트리거**: 키보드(Delete/Backspace) + 호버 ⋮ 메뉴(위와/아래와 합치기)
- **병합 가드**: 첫 번째 블록에서 Backspace, 마지막 블록에서 Delete 시 병합 요청을 보내지 않음 (서버 호출 자체 차단)
- **병합 시 태그 처리**: 두 블록 중 더 높은 값 또는 사용자 선택
- **분할 시 태그·화자 상속**: 앞 블록이 상속, 뒤 블록은 "미지정" 상태로 시작
- **발견성 보완**: 편집 진입 시 하단에 잠깐 치트라인 `Enter=확정 · Shift+Enter=줄바꿈 · Ctrl+Enter=분할 · Esc=취소` 표시

#### 호버 ⋮ 메뉴
- 위와 합치기
- 아래와 합치기
- (향후) 화자 태깅, 블록 삭제 등
- **분할은 포함 안 함**

#### 일괄 치환
- **진입**: 툴바 "치환" 버튼 → 전사 영역 **상단 고정 바** 등장 (VS Code 스타일)
- **동작**: 일괄 확정 + 순차 확인 모두 지원 (표준 검색/치환 패널)
- **옵션 플래그**:
  - 대소문자 구분
  - 단어 단위 일치
  - **잠금 구간 제외** (기본 ON) — 사용자가 수동 수정한 블록은 의도적 문구이므로 치환 대상에서 제외
- **정규식**: 미지원 (개인용 앱엔 과함)
- **Undo 범위**: 치환 전체가 하나의 Undo 단위
- **영향 범위**: 본문 텍스트만, 블록 경계·타임스탬프·중요도·화자 슬롯 불변

#### 메타데이터 영역
- **필드 구성**: 제목 / 날짜(자동) / 소요시간(자동) / 언어 / 참여자 / 장소
- **기본 상태**: 펼침 (이전 단계와 연속성, 편집 빈도 높음)
- **빠른 시작 경로 미입력 강조**:
  - 5단계 진입 시 빈 필드에 **옅은 노란 배경** 으로 자연스러운 시선 유도 (강제 배너 없음)
  - 7단계(전송) 직전에 한 번만 확인 ("참여자가 비어 있습니다. 계속하시겠어요?")
- **참여자 입력**: **주소록 기반 태그 입력** + inline creation (주소록이 비어 있어도 현장에서 새 항목 추가)
- **장소 입력**: 참여자와 동일 원칙 (주소록 + inline creation)
- **저장 시점**: 필드 **blur 시 자동 저장** (명시적 저장 버튼 없음)

### 6단계 (요약 & 액션 아이템) 상세 설계

#### 요약 생성 타이밍
- 5단계에서 [다음 단계] 클릭 시 **요약 생성 여부 선택 모달** 표시:
  ```
  AI 요약을 생성할까요?

  [ 요약 생성 ]    [ 건너뛰고 전송으로 ]

  건너뛰기: 요약 없이 전사 원본만 저장·전송합니다.
  ```
- **[요약 생성]**: 기존 플로우 — 로딩 오버레이(스피너 + "AI 요약 생성 중…") 후 6단계 진입
- **[건너뛰고 전송으로]**: 6단계 스킵, **5→7 직행**
  - 7단계에서 `summary_markdown`이 null이면 "요약 없음" 표시
  - Slack 메시지에서 "핵심 요약" 섹션 생략, 전사 원본 .md만 첨부
  - .md export에도 요약 섹션 없이 전사 원본만 포함
- **진행 표시** (요약 생성 선택 시): 전체 화면 로딩 오버레이 — 스피너 + "AI 요약 생성 중…"
- 입력: 5단계에서 중요도 **상+중** 으로 태깅된 블록 ("핵심만 보기" 범위와 동일)

#### 요약 템플릿 구조 (실제 회의록 샘플 기반)
```markdown
# [날짜(요일)] [회의 제목]

## 회의 개요
- 회의 시간: [시작] ~ [종료] ([소요 시간])
- 장소: [장소]
- 참여자: [목록]
  - [특이사항: 지각, 부재, 중도 참여 등]
- 언어: [언어]

## 주요 논의 사항 & F/U 필요 요소

### 1. [주제명]
**주요 논의**
- 논의 내용 bullet
- 논의 내용 bullet

**F/U 필요 사항**
- [@담당자] 할 일 내용 (~기한)

### 2. [주제명]
**주요 논의**
- ...

**F/U 필요 사항**
- ...

---

## 기타 메모
- (주제에 속하지 않는 부가 논의, 공지 등)
```

- **회의 개요**: 2단계 메타데이터를 구조화된 데이터로 삽입 (AI 추출 아님)
- **주제별 그룹핑**: AI가 전사에서 주제를 식별, 각 주제 안에서 "논의 + F/U" 분리
- **개별 회의록(전사 상세)은 미포함**: 요약은 요약답게, 상세는 전사 원본 및 첨부 .md의 역할

#### 요약 편집
- **블록 단위**: `##` / `###` 헤딩 기준 분리
  - 회의 개요 = 1블록
  - 각 주제(### N. 주제명) = 1블록
  - 기타 메모 = 1블록
- **더블클릭/Enter 편집** (전사 블록과 동일 인터랙션 패턴)
- **WYSIWYG 라이브 렌더링** (Notion/Slack 스타일)
  - 마크다운 원문을 보지 않음, 마크다운 문법은 입력 단축키 역할
  - `**텍스트**` → 즉시 굵게, `- ` → 즉시 불릿, `## ` → 즉시 헤딩
  - 마크다운 모르는 사용자도 일반 텍스트 수정 가능
- **서식 치트라인**: 첫 편집 진입 시 `**굵게** · - 목록 · ## 제목` 팁 한 번 표시, 이후 숨김
- **블록 병합·분할**: Shift+Enter(분할) / Backspace·Delete(병합) — 전사 블록과 동일
- **재요약 버튼**: 전체 재생성

#### 액션 아이템 (F/U 자동 집계)
- 요약 본문의 각 주제 안 "F/U 필요 사항"에서 **자동 추출하여 6단계 하단에 일괄 표시**
- **불릿 리스트** 형태 (체크박스·테이블 아님, 부담 최소화)
```
전체 F/U 요약
• 보고서 초안 작성 @김OO ~04/20 — 주제 1
• 디자인 리뷰 요청 @박OO ~04/18 — 주제 2
• 회의실 예약 확인 — 주제 2
[+ 항목 추가]
```
- 주제별 F/U(맥락 안에서 확인) + 전체 집계(한눈에 확인) 두 관점 모두 제공
- 사용자가 수동으로 항목 추가·수정·삭제 가능
- **편집 진입**: 더블클릭 or ↑/↓ 포커스 → Enter (전사 블록과 동일 패턴)
- **확정/취소**: Enter(편집 중) / 밖 클릭 / Esc
- **담당자 입력**: `@` 입력 시 **참여자 주소록 드롭다운** 등장, 목록 외 자유 입력 허용
- **항목 삭제**: 호버 시 우측 ✕ 버튼

#### 전사 참조
- **접힌 패널** (읽기 전용), 필요할 때만 펼침
- 편집 불가 — 전사 수정이 필요하면 5단계로 돌아감

#### 앱 전체 편집 패턴 통일
| 대상 | 블록 단위 | 더블클릭/Enter 편집 | Shift+Enter 줄바꿈 | Ctrl+Enter 분할 | 병합 |
|------|----------|-------------------|-------------------|-----------------|------|
| 전사 (3·5단계) | 문장 | ✓ | ✓ | ✓ | ✓ |
| 요약 (6단계) | 단락(##/### 기준) | ✓ | ✓ | ✓ | ✓ |
| 액션 아이템 (6단계) | 불릿 항목 | ✓ | — | — | — |

### 7단계 (전송 & 저장) 상세 설계

#### 화면 구성
```
[ [🏠] wizard stepper                        ]

📤 전송 & 저장

[ Slack 전송 ]
  채널: #team-meeting ▼  (템플릿에서 자동 채움 가능)
  전송 방식: (●) 새 메시지  (○) 스레드 답장
  (스레드 답장 선택 시 → 최근 메시지 목록에서 선택)
  [ 미리보기 ▼ ]  ← 접힌 패널

[ 로컬 저장 ]
  저장 경로: C:\...\meetings\  [폴더 선택]
  파일명: 주간_팀_미팅_20260416.md  (자동 생성)

- **저장 경로 선택**: [폴더 선택] 클릭 → **OS 폴더 선택 창** (`window.showDirectoryPicker()`, File System Access API) 호출
  - Chrome/Edge 지원 (이 앱은 Web Speech API로 인해 Chrome 전용이므로 호환성 문제 없음)
  - 사용자가 폴더를 선택하면 해당 경로가 표시됨
  - 선택한 디렉토리 핸들을 사용하여 서버 경유 없이 **브라우저에서 직접 .md 파일 저장** 가능
  - 또는 선택한 경로를 서버에 전달하여 서버 측에서 저장 (로컬 앱이므로 서버도 같은 머신)
- 변경된 경로는 **해당 세션에만 적용** (설정 기본값은 변경하지 않음)
- 기본값은 설정 화면의 `export_path`에서 가져옴
- 설정 화면에서도 동일한 [폴더 선택] 버튼으로 기본 경로 설정 가능

[ JSON 히스토리 ]
  ✓ 자동 저장됩니다

──────────────────────────────
☑ Slack 전송    ☑ .md 저장    ✓ JSON 히스토리 (자동)
[← 이전 단계]                [ 📤 실행 ]
```

#### Slack 채널 & 스레드 선택
- **채널 선택**: Slack Bot API로 봇이 참여한 채널 목록 드롭다운
- **스레드 선택 (MVP 지원)**:
  - 기본: "새 메시지"
  - "스레드 답장" 선택 시: 해당 채널의 **최근 메시지 20건**을 **메시지 카드 목록**으로 표시
  - 사용자가 카드 클릭으로 스레드 대상 선택
  - 향후: 검색 + 템플릿 자동 매칭 확장 가능
- **채널은 템플릿에 저장 가능**: 2단계 템플릿 구조에 `slack_channel` 필드 포함

#### 스레드 선택 — 메시지 카드 UI
- raw 텍스트가 아닌 **구조화된 카드** 형태로 표시하여 어떤 메시지인지 즉시 식별 가능
```
┌──────────────────────────────────────────────┐
│ 🤖 meetingRecorder           04/18(금) 14:32  │
│ [04/18(금) 14:00 정기 회의] 오늘 진행된 회…    │
│                                💬 3개 답글     │
├──────────────────────────────────────────────┤
│ 👤 김OO                      04/17(목) 10:15  │
│ 다음 주 일정 공유드립니다. 월요일…              │
│                                💬 7개 답글     │

- **스레드 모드 전환 시 즉시 로딩**: "스레드 답장" 선택 시 현재 선택된 채널의 메시지를 **즉시 fetch** (채널 재선택 없이도 로딩)

### 4단계 (Whisper 후처리) 상세 설계

#### 처리 내용
1. 오디오 파일을 로컬 Whisper medium으로 재처리
2. 사용자 수정 잠금 구간과 Whisper 결과를 타임스탬프 기반 병합
3. AI 중요도 1차 태깅 (자동 스킵 아닌 경우)

#### Whisper 처리 선택 화면 (처리 시작 전)
```
[ [🏠] wizard stepper                         ]
[                                              ]
[   Whisper로 전사 정확도를 높일 수 있습니다     ]
[   (예상 처리 시간: ~5분)                      ]
[                                              ]
[   [ Whisper 처리 시작 ]    [ 건너뛰기 ]       ]
[                                              ]
[   건너뛰기: Web Speech 결과만으로 편집을       ]
[   진행합니다. 정확도가 낮을 수 있습니다.        ]
[                                              ]
```
- **[Whisper 처리 시작]**: 아래 진행률 화면으로 전환
- **[건너뛰기]**: Whisper 없이 Web Speech 블록 그대로 5단계로 이동. AI 태깅은 별도 실행 (Whisper와 독립)
- **업로드 경로에서는 건너뛰기 불가** — Whisper가 유일한 전사 소스이므로 버튼 숨김, 자동으로 처리 시작

#### 화면 구성 — 처리 진행률
```
[ [🏠 비활성] wizard stepper                  ]
[                                              ]
[         🔄 Whisper 처리 중… (43%)            ]
[         ████████░░░░░░░░░ 43%               ]
[         예상 잔여 시간: ~3분                  ]
[                                              ]
[         ✓ 오디오 파일 로드 완료               ]
[         ✓ Whisper 전사 처리 중…              ]
[         ○ 사용자 수정 병합                    ]
[         ○ AI 중요도 태깅                      ]
[                                              ]
```
- 단계별 체크리스트로 현재 진행 단계 시각화
- 향후 불편함 느끼면 "전사 미리보기(옵션 B)" 업그레이드 검토

#### 처리 중 이탈
- **이탈 차단** — 4단계에서 다른 단계 이동 불가, 완료까지 대기
- beforeunload 경고 유지 (처리 중 창 닫기 방지)
- 향후 대기가 부담스러우면 백그라운드 처리 + 브라우저 알림으로 업그레이드 가능

#### 처리 완료 후 전환
- **수동 전환** — "✓ 처리 완료! [다음 단계 →]" 버튼 표시, 사용자가 클릭해야 이동
- 완료 시 브라우저 탭 타이틀에 "✓ 처리 완료" 표시 (다른 탭에 있어도 인지 가능)

#### 처리 실패 대응
- 오류 메시지 + **재시도 버튼** + **"Web Speech 결과만으로 계속"** 옵션
- Web Speech만으로도 5단계 편집은 가능하므로 전체 플로우가 막히지 않음
- 재시도해도 실패 시 Web Speech 결과로 진행

### 2단계 (회의 정보 입력) 상세 설계

#### 레이아웃 (단일 폼, 옵션 B)
```
[ [🏠] wizard stepper                        ]

[ 템플릿: (선택 안 함 ▼) ]  ← 선택하면 자동 채움
[ 제목: _________________ ]  ← 필수
[ 참여자: _______________ ]  ← 선택
[ 장소: _________________ ]  ← 선택
[ 언어: (한국어 ▼)        ]

──────────────────────────────
[ 입력 소스: [ 🎙 실시간 녹음 | 📁 파일 업로드 ] ]

(📁 선택 시 드롭존 등장)
┌─────────────────────────────────────┐
│   📁 파일을 드래그하거나 클릭해 선택  │
│   지원 형식: .webm .mp3 .wav .m4a   │
└─────────────────────────────────────┘

[ [← 이전 단계]             [다음 단계 →] ]
```

#### 템플릿 → 수정 UX
- 템플릿 미선택 = 빠른 시작 (제목만 입력 후 바로 진행)
- 템플릿 선택 시: **토스트 + 파란 배경** 조합
  - **토스트** (하단, 3초 후 사라짐): "'주간 팀 미팅' 기본값을 불러왔습니다. 수정해도 템플릿은 변경되지 않습니다."
  - **파란 배경**: 템플릿에서 자동 채움된 필드에 옅은 파란 배경 → 사용자가 수정하면 배경 사라짐
  - **포커스 자동 이동**: 템플릿 선택 후 제목 필드로 커서 이동 (확인·수정 흐름 유도)
- 2회차부터 토스트는 짧은 버전("기본값을 불러왔습니다") 가능

#### 입력 소스 선택
- **세그먼트 컨트롤**: [ 🎙 실시간 녹음 | 📁 파일 업로드 ]
- 기본값: 실시간 녹음

#### 마이크 권한
- **2단계 진입 시 `navigator.permissions.query`로 상태 체크**
  | 권한 상태 | 동작 |
  |----------|------|
  | `granted` (이미 허용) | 조용히 통과 |
  | `prompt` (미설정, 첫 사용) | 브라우저 권한 팝업 트리거 |
  | `denied` (차단됨) | 안내 패널 + 브라우저 설정 경로 안내 |
- 브라우저가 origin 단위로 권한 캐시 → **실질적으로 첫 1회만 팝업**

#### 파일 업로드 (2단계에서 완료)
- 파일 업로드 선택 시 드롭존이 세그먼트 아래에 등장
- 파일 첨부 완료 후 "다음 단계" → **3단계 스킵** → 4단계(Whisper)로 직행
- 지원 형식: .webm .mp3 .wav .m4a

#### 세션 생성
- "다음 단계" 클릭 시 서버에 세션 JSON 생성 (메타데이터 + 오디오 파일 경로)
- 크래시 복구 시 메타데이터 보존 가능 (예외 처리 섹션 참조)

### Wizard Stepper 경로별 동작

#### 전체 단계 맵
| 번호 | 라벨 | 핵심 역할 |
|------|------|----------|
| 1 | 홈 | 새 회의 시작, 히스토리 접근 |
| 2 | 회의 정보 입력 | 템플릿/빠른 시작, 모드 선택, 파일 첨부 |
| 3 | 녹음 & 실시간 전사 | 마이크 녹음 + Web Speech 실시간 전사 |
| 4 | Whisper 후처리 | 오디오 → Whisper 재처리 + AI 태깅 |
| 5 | 회의록 편집 | 전사 수정, 중요도 태깅, 일괄 치환 |
| 6 | 요약 & 액션 아이템 | Claude 요약, 불릿 액션 아이템 |
| 7 | 전송 & 저장 | Slack 전송, 로컬 저장 |

#### 경로별 흐름
- **녹음 경로**: 1→2→3→4→5→6→7 (전 단계 순서대로)
- **업로드 경로**: 1→2→[3 스킵]→4→5→6→7

#### stepper 표시 (업로드 경로)
```
● ── ● ── ○ ── ◉ ── ○ ── ○ ── ○
1    2   [3]   4    5    6    7
홈  정보  건너뜀 처리  편집  요약  전송
```
- 3단계는 회색 점 + "건너뜀" 표시, 번호 체계는 양쪽 동일

#### 반응형 Stepper

| 화면 크기 | 표시 방식 |
|----------|----------|
| 데스크탑 (768px 이상) | 기존 전체 dot + 라벨 표시 |
| 모바일 (768px 미만) | 축약형: 홈 아이콘 + 현재 단계 번호/라벨만 |

**데스크탑 (기존)**:
```
[🏠]  ● ── ● ── ● ── ◉ ── ○ ── ○ ── ○
      1    2    3    4    5    6    7
```

**모바일 축약형**:
```
[🏠]                    ◉ 4/7 편집
```
- 홈 아이콘은 유지 (이탈 동선 보장)
- 가운데에 현재 단계 dot + "N/7 라벨"
- 전체 진행률은 숫자로 인지 ("4/7" = 7단계 중 4단계)
- 업로드 경로에서 3단계 스킵 시: "4/7 처리" (총 단계 수는 7 고정, 건너뛴 단계도 포함)

#### Wizard 네비게이션 설계

##### 하단 네비게이션 바
```
[← 이전 단계]                    [다음 단계 →]
```
- 2단계부터 하단 좌측에 [← 이전 단계] 버튼 표시
- 1단계(홈)에서는 이전 버튼 없음
- **두 버튼은 동일한 디자인 톤의 쌍**으로 구성:

| 요소 | 이전 단계 | 다음 단계 |
|------|----------|----------|
| 스타일 | **Secondary 버튼** | **Primary 버튼** |
| 클래스 | `bg-bg-subtle text-text rounded-lg px-5 py-3 text-[15px] font-medium hover:bg-bg-hover` | `bg-primary text-white rounded-lg px-5 py-3 text-[15px] font-semibold hover:bg-primary-hover` |
| 아이콘 | `ArrowLeft` (좌측) | `ArrowRight` (우측) |
| 위치 | 좌측 정렬 | 우측 정렬 |
| 비활성 | `opacity-30 cursor-not-allowed` | `opacity-30 cursor-not-allowed` |

- 이전 버튼은 **텍스트 링크가 아닌 Secondary 버튼** — 다음 버튼과 크기·형태가 대칭
- 두 버튼을 `flex justify-between` 컨테이너로 감싸 좌우 정렬
- 이전 버튼이 없는 경우(1단계)에는 `justify-end`로 다음 버튼만 우측 표시

##### 홈 아이콘 (Stepper 좌측)
```
[🏠]  ● ── ● ── ● ── ◉ ── ○ ── ○ ── ○
      1    2    3    4    5    6    7

- Wizard Stepper 좌측에 홈 아이콘(lucide `Home`) 상시 배치

- 세션을 중단 상태로 보존하고 홈으로 이동 → 홈의 복구 배너에서 이어가기 가능
- 녹음 중(상태 2)이면 모달 문구 변경: **"녹음을 중지하고 홈으로 돌아가시겠어요?"** → 녹음 자동 중지 후 이동
- 4단계(Whisper 처리 중)에서는 **비활성** (처리 완료까지 대기)

##### [← 이전 단계] 버튼 단계별 동작

| 현재 위치 | 클릭 시 이동 | 조건/비고 |
|----------|------------|----------|
| 1(홈) | 버튼 없음 | 홈이 시작점 |
| 2(정보 입력) | 1(홈)으로 | 세션 미생성이면 바로 이동. 세션 생성 후면 확인 모달("홈으로 돌아가시겠어요? 진행 상태는 저장됩니다.") → **세션 보존** (복구 배너에서 이어가기/삭제) |
| 3(녹음) | 2(정보 입력)으로 | **녹음 중(상태 2)이면 비활성** — 중지(상태 3) 후에만 이동 가능 |
| 4(Whisper) | **버튼 숨김** | 처리 중 이탈 차단 (기존 규격 유지) |
| 5(편집) | 3(녹음)으로 | 녹음을 추가하고 싶은 경우 돌아갈 수 있음. 3단계 post_recording 상태로 진입 |
| 6(요약) | 5(편집)으로 | 태깅 수정·전사 재편집 가능 |
| 7(전송) | 6(요약)으로 | 요약 재확인 가능 |

업로드 경로 추가 규칙:
| 현재 위치 | 업로드 경로에서 이전 |
|----------|-------------------|
| 4(Whisper) → 이전 | **2(회의 정보)로** (3 스킵) |

##### 6→5→6 재진입 시 재요약 안내
- 6단계에서 [← 이전]으로 5단계에 돌아가 편집 후, 다시 [다음 단계 →]를 누르면:
- 확인 모달: **"태깅이나 전사를 수정했습니다. 요약을 다시 생성할까요?"** → [재요약] / [기존 요약 유지]
- [재요약] 선택 시 기존과 동일한 요약 생성 플로우 실행
- [기존 요약 유지] 선택 시 이전 요약 그대로 6단계 표시

##### 브라우저 뒤로가기(Alt+←) 동기화
- React Router로 단계별 URL을 관리하므로 `popstate` 이벤트에서 위와 동일한 이동 규칙 적용
- 차단 상황(녹음 중, 4단계 처리 중)에서는 `history.pushState`로 이탈 방지
- 브라우저 뒤로가기와 [← 이전 단계] 버튼의 동작이 항상 동일하도록 통일

## 예외 처리 & 리스크 대응 기획

### 오디오 보존 (핵심 설계)

#### 실시간 오디오 스트리밍
- **MediaRecorder → 로컬 FastAPI 서버로 WebSocket 스트리밍**
- 백엔드가 이미 Whisper·API 중계·파일 시스템 접근 역할을 하므로 자연스러운 확장
- **5초 청크 단위**로 서버에 append → 최악의 경우도 손실 5초 이내
- 브라우저 크래시·강제 종료 시에도 오디오는 이미 디스크에 존재
- "위험 감지" 개념 불필요 — **항상 저장됨**

#### 세션 구조 (메타데이터 + 오디오)
- 2단계 회의 정보 입력 완료 시 서버에 **세션 생성**
- 세션 파일에 메타데이터(제목/참여자/장소/언어/시작시각) + 오디오 청크 디렉토리 경로 포함
```
data/sessions/{session_id}/
  session.json                   ← 메타데이터 + 블록
  chunks/
    chunk_000.webm               ← 오디오 청크 (개별 파일)
    chunk_001.webm
    ...
```
- 오디오 청크는 개별 파일로 저장 (WebM 포맷 무결성 보장), Whisper 처리 직전 ffmpeg로 병합
- 크래시 복구 시 메타데이터도 함께 복원
- 빠른 시작 경로(제목만 입력)로 시작한 경우: 제목만 복구, 나머지는 빈 상태

#### 복구 UX
- 홈 화면(1단계) 배너로 진행 중인 회의 카드 표시 (위 "복구 배너" 참조)
- **복구 = 기존 세션 재개** — 새 세션을 생성하지 않고, 기존 세션 ID 그대로 사용
- **세션 status에 따라 진입 단계가 달라짐**:
  | 세션 status | 복구 진입 단계 | 상태 |
  |-------------|--------------|------|
  | `recording` / `post_recording` | **3단계** (녹음) | post_recording으로 진입, **기존 전사 블록을 서버에서 로드하여 표시**, 재개/다음 선택 가능 |
  | `processing` | **4단계** (Whisper) | 처리 재시작 |
  | `editing` | **5단계** (편집) | 편집 이어서 진행 |
  | `summarizing` | **6단계** (요약) | 요약 이어서 진행 |
  | `idle` | **2단계** (정보 입력) | 세션 생성만 되고 녹음 시작 전 |
- 메타데이터(제목·참여자·장소 등)는 세션에 이미 저장되어 있으므로 별도 복원 로직 불필요
- **3단계 복구 진입 시 필수 동작**:
  1. `GET /api/sessions/{id}`로 세션 데이터 로드 (메타데이터 + 블록)
  2. 기존 전사 블록을 전사 영역에 표시 — 사용자가 이전 녹음 내용을 즉시 확인 가능
  3. 상단에 **복구 안내 배너**: "이전 녹음이 복원되었습니다. 이어서 녹음하거나 다음 단계로 진행하세요." (토스트가 아닌 인라인 배너, 닫기 가능)
  4. `post_recording` 상태로 진입 → [녹음 재개] 또는 [다음 단계] 선택 가능
- **"새 회의 시작"은 기존 세션 삭제 후에만 가능** — 동시 세션 방지 규칙과 충돌하지 않음

### 백그라운드 탭 대응

#### 문제
- Chrome이 비활성 탭을 throttle → Web Speech API 중단, JS 타이머 지연
- 다른 탭 사용 후 돌아오면 실시간 전사가 멈춰있는 현상 발생

#### 안전망 (아키텍처 수준)
- Web Speech는 실시간 "미리보기" 전용, **최종 전사는 Whisper가 담당**
- MediaRecorder는 OS 오디오 레벨에서 캡처 → 탭 throttle 영향 적음
- 오디오는 5초 청크로 서버에 계속 저장됨 → **데이터 손실 없음**

#### 선제 대응 (5종)
| 대응 | 설명 | 목적 |
|------|------|------|
| **① 무음 오디오 재생** | `AudioContext`로 무음 버퍼 루프 재생, Chrome이 "미디어 재생 중" 탭으로 인식 → throttle 완화 | 근본 대응 |
| **② 탭 복귀 시 자동 재연결** | `visibilitychange` 이벤트 → Web Speech 재시작 + 타이머 보정 | 복구 |
| **③ 조건부 토스트 경고** | Web Speech가 **실제로 중단·재연결된 경우에만** "일부 전사가 누락됐을 수 있습니다" 표시. 정상 복귀 시에는 표시 안 함 | 사용자 고지 (불필요한 불안 방지) |
| **④ 타이머 보정** | `Date.now()` 기반 실제 경과 시간 계산 (`setInterval` 의존 X) | 타이머 정확도 |
| **⑤ Web Worker** | 오디오 청크 전송 로직을 Web Worker로 분리 → 메인 스레드 throttle 영향 감소 | 스트리밍 안정성 |

- ①의 성능 영향: CPU < 0.1%, 메모리 수 KB — 사실상 무부하 (Google Meet·Zoom Web 등 동일 기법 사용)
- ③ 원칙: **발생하지 않은 리스크에 대해 사용자를 불안하게 하지 않음**

### 3단계 이탈 / 사고 대응 (요약)
| 시나리오 | 대응 |
|---------|------|
| 마이크 권한 거부 | 2단계에서 미리 권한 요청 |
| 창 닫힘 / 새로고침 | `beforeunload` 경고 |
| 마이크 끊김 | 경고 + 상태 3(post-recording) 자동 전환 |
| 네트워크 끊김 (Web Speech) | 백그라운드 자동 재시도, 녹음은 계속 |
| 배터리 / 절전 | 진입 시 가벼운 팁만 |

### 4단계 처리 실패 대응
- 오류 메시지 + 재시도 버튼 + "Web Speech 결과만으로 계속" 옵션
- 전체 플로우가 막히지 않도록 fallback 보장

- 기존 wizard 단계 재활용, 추가 구현 최소

- **제공 위치**: 히스토리 상세 하단 `[🗑 삭제]` 버튼
- **삭제 플로우**:
  1. [삭제] 클릭
  2. 확인 모달: **"이 회의록을 삭제할까요? 전사·요약·오디오 파일이 모두 삭제됩니다."**

     - [삭제] / [취소]
  3. 성공: 목록으로 돌아감 + 토스트 "삭제 완료"
- **삭제 범위**: meeting JSON + 병합 오디오 + export .md 모두 삭제
- **Slack 메시지**: 삭제하지 않음 (별도 "메시지 삭제" 기능으로 분리)

| 장소 주소록 | 추가/수정/삭제 | 동일 |
| Slack 연동 | Bot Token 설정, 연결 테스트 | |
| Claude API | API 키 설정 | 마스킹 표시, 백엔드 저장 |
| Whisper | 모델 선택 (tiny~large) | 모델별 처리시간·정확도 안내 |
| 마이크 민감도 | 기본 민감도 설정 (0.5x~3.0x) | 3단계 녹음 시 초기값으로 사용, 슬라이더 UI |
| Slack 인사 문구 | 커스터마이징 (여러 줄 + 이모티콘) | textarea 입력, `:emoji:` 문법 지원, Slack mrkdwn `\n`으로 전송 |
| 저장 경로 | .md 기본 저장 경로 | |
| 요약 템플릿 | MVP 고정, 향후 커스터마이징 | |

#### API 키 보안
- API 키는 마스킹 표시 (`sk-ant-●●●●●●●●●●XYZ`)
- [변경] 클릭 시 새 키 입력 → 저장
- 키는 서버 환경변수 또는 로컬 설정 파일에 저장
- 프론트엔드에 키 원문 노출 안 함 (백엔드 경유)

#### Whisper 모델 선택 가이드
| 모델 | 처리 시간 (1시간 기준) | 정확도 | 안내 |
|------|---------------------|--------|------|
| tiny | ~1분 | 낮음 | 빠른 미리보기용 |
| base | ~2분 | 보통 | 가벼운 회의용 |
| small | ~3분 | 양호 | 균형 |
| **medium** | ~5-8분 | 높음 | **추천 (기본값)** |
| large | ~15분+ | 최고 | 정확도 최우선 시 |

#### Claude API 비용 참고
- 회의 1건당 예상 비용: ~$0.025 (태깅 Haiku + 요약 Sonnet)
- 월 12회 기준: ~$0.30/월
- 프롬프트 캐싱 적용 시 추가 절감 가능
