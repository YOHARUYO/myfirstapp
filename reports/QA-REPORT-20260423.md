### 7-4. Slack 봇 채널 이슈 — 해결 ✅

**증상:** 사용자가 `#new_bc통합` 채널에 meetingrecorder 봇을 추가했으나, 프론트엔드 채널 선택 UI에서 `#전략팀`만 표시. 새로고침해도 변화 없음.

**조사 과정:**
1. `.env`와 `settings.json`의 Slack 토큰 일치 확인 (동일 토큰)
2. Slack API 직접 호출 → `#new_bc통합`은 **2페이지**에 위치, `is_member=True` 정상
3. 백엔드 코드에 페이지네이션 로직 존재 확인
4. 독립 Python 스크립트로 동일 로직 실행 → 3개 채널 정상 반환
5. 하지만 서버 API는 1개만 반환 → 서버 프로세스 조사

**근본 원인:** `uvicorn main:app --reload --port 8000` 명령이 **2개 동시 실행** (PID 45784, 47180). 이전 세션에서 서버를 종료하지 않고 새로 시작하여 포트 8000에 2개 프로세스가 바인딩. 요청이 옛날 프로세스로 라우팅되면서 최신 코드(페이지네이션 포함)가 반영되지 않음.

**해결:** 모든 Python 프로세스 강제 종료 후 단일 인스턴스로 재시작 → 3개 채널 정상 반환:
- `#전략팀` (C07AY2YR9B9)
- `#new_bc통합` (C09RF4DA82W)
- `#전략팀4인` (C09SFKABKAP)

**교훈:** 서버 재시작 시 기존 프로세스 완전 종료 확인 필수. `--reload` 모드에서 자식 프로세스가 남을 수 있음.
